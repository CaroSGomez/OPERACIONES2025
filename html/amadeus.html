¿Como tomar una reserva de cero en Amadeus?

Paso 1: Agregar nombres a la reserva
El indicador de nombre es: NM1
El orden siempre debe ser Apellido/nombre
El agregar los nombres de los pasajeros en primer lugar facilita el proceso de guardar lugares en la reserva

Ejemplo de nombre: NM1GOMEZ/CAROLINA
Paso 2: Buscar la disponibilidad deseada
El indicador de busqueda es: AN
El orden siempre debe ser FECHA/RUTA/CIA
Ejemplo de busqueda para el 26/09
Saliendo de Aeroparque a Bariloche por Jetsmart: AN26SEPAEPBRC/AJA
Paso 3: Guardar lugares
En el smart podemos tomar los segmentos dando click sobre la clase que queremos
El proceso con comandos seria el siguiente: SS1L1
SS: Vender
1: Cantidad de pasajeros
L: Clase
1: Segmento de vuelo
Paso 4: Guardar datos basicos de reserva
Lo primero que debemos grabar en el pnr para poder cerrarlo es la informacion de la empresa.
El comando que guarda el telefono de la empresa es: AP
El segundo dato obligatorio es el tiempo limite.
El comando que guarda el plazo es: TKTL
Paso 5: Guardar datos de pasajeros
Para guardar el telefono del pasajero debemos usar el comando: SRCTCMYYHK1-xxxxxxxx
Ejemplo de telefono para el pax 1: SRCTCMYYHK1-541168554866
Para guardar el mail del pasajero debemos usar el comando: SRCTCEYYHK1-xxxxxxxx
Ejemplo de mail para el pax 1: SRCTCEYYHK1-gomez//atrapalo.com.ar/P1
Por ultimo agregaremos el DNI del pax usando el comando:
SRFOIDYYHK1-NIxxxxxxxx/Px
Si quisieramos cargar un pasaporte debemos cambiar las siglas NI por PP.
SRFOIDYYHK1-PPxxxxxxxx/Px
Ejemplo de DNI para el pax 1: SRFOIDYYHK1-NI34587951/P1
Si bien ya pudimos generar el pnr correspondiente a la reserva de cero, ahora falta la cotizacion y guardado de tarifa.
Paso 6: Cotizar y guardar tarifa
Tenemos diferentes maneras de cotizar de acuerdo a lo que deseamos.
FXR: Este comando cotiza la clase tomada. No guarda tarifas
FXP: Este comando cotiza la clase tomada y guarda la tarifa
FXB: Se usa para que el sistema nos muestre la opcion más economica disponible. Permite guardar la tarifa
Paso 7: Cerrar la reserva
Una vez completos los datos de pasajeros, itinerario, datos basicos y plazo, debemos firmar todo el proceso y finalmente cerrarlo para guardarlo.
El comando para firmar es: RFIniciales
El comando para cerrar es: ER

Void

Para poder voidear una reserva, tenemos que chequear el estatus del ticket
El unico estatus que permite voidear es: OPEN

Status de Ticket:

OPEN Ticket abierto para gestión
- Permite voidear
CHKIN Ticket con check-in hecho
- No permite voidear
ARTP Ticket en control de aeropuerto
- No permite voidear
EXCH Ticket cambiado
- No permite voidear

El comando de void es: TRDC/L(nrodelinea)
Luego de voidear, el sistema nos devolverá las siguientes respuestas:

Vamos a volver a desplegar el ticket para chequear el estatus:
Comando para desplegar el ticket: TWD/L(nro de linea donde esta el ticket)
Es importante destacar que el comando SOLO void el ticket, no elimina los segmentos.
Es mala práctica dejar segmentos/tramos inactivos.
Siempre vamos a limpiar el itinerario si no vamos a emitir un ticket.
El comando para limpiar el pnr es: XI
El gds nos informa que el itinerario ha sido limpiado, y el ticket ha sido voideado.
Veremos solo el nombre en el pnr.

¿Como tomar una reserva de cero en Galileo?

Paso 1: Agregar nombres a la reserva
El indicador de nombre es: N.
El orden siempre debe ser Apellido/nombre
El agregar los nombres de los pasajeros en primer lugar facilita el proceso de guardar lugares en la reserva

Ejemplo de nombre: N.GOMEZ/CAROLINA
Paso 2: Buscar la disponibilidad deseada
El indicador de busqueda es: A
El orden siempre debe ser FECHA/RUTA/CIA
Ejemplo de busqueda para el 26/09
Saliendo de Aeroparque a Jujuy por Aerolineas Argentinas: A26SEPAEPJUJ/AR
Paso 3: Guardar lugares
En el smart podemos tomar los segmentos dando click sobre la clase que queremos
El proceso con comandos seria el siguiente: 01Y3
0: Vender
1: Cantidad de pasajeros
Y: Clase
3: Segmento de vuelo
Paso 4: Guardar datos basicos de reserva
Lo primero que debemos grabar en el pnr para poder cerrarlo es la informacion de la empresa.
El comando que guarda el telefono de la empresa es: CM/
El segundo dato obligatorio es el tiempo limite.
El comando que guarda el plazo es: T.T/

Paso 5: Guardar datos de pasajeros
Para guardar el telefono del pasajero debemos usar el comando: SI.Px/SSRCTCMYYHK1/xxxxxxxx
Ejemplo de telefono para el pax 1: SI.P1/SSRCTCMYYHK1/541168554866
Para guardar el mail del pasajero debemos usar el comando: SI.Px/SSRCTCEYYHK1/xxxxxxxx
El @ se carga con //
El _ se carga con ..
Ejemplo de mail para el pax 1: SI.P1/SSRCTCEYYHK1/gomez//atrapalo.com.ar
Si bien ya pudimos generar el pnr correspondiente a la reserva de cero, ahora falta la cotizacion y guardado de tarifa.
Paso 6: Cotizar y guardar tarifa
Tenemos diferentes maneras de cotizar de acuerdo a lo que deseamos.
FQ: El comando para cotizar la clase tomada:
FQBB: Se usa para que el sistema nos muestre la opcion más economica disponible
FQ/BAG: Este comando cotiza con equipaje para despachar en bodega
FQBB++-BUSNS: Este comando muestra tarifas Business
Paso 7: Cerrar la reserva
Una vez completos los datos de pasajeros, itinerario, datos basicos y plazo, debemos firmar todo el proceso y finalmente cerrarlo para guardarlo.
El comando para firmar es: R.Iniciales
El comando para cerrar es: ER

Void

Para poder voidear una reserva, tenemos que chequear el estatus del ticket
El unico estatus que permite voidear es: OPEN

Status de Ticket:

OPEN Ticket abierto para gestión
- Permite voidear
CHKIN Ticket con check-in hecho
- No permite voidear
ARTP Ticket en control de aeropuerto
- No permite voidear
EXCH Ticket cambiado
- No permite voidear

El comando de void es: TRV/(nrodetkt)

Luego de voidear, el sistema nos devolverá las siguientes respuestas:

Vamos a ignorar y volver a desplegar el ticket para chequear el estatus:
El comando para ignorar es: IR
Es importante destacar que el comando SOLO void el ticket, no elimina los segmentos.
Es mala práctica dejar segmentos/tramos inactivos.
Siempre vamos a limpiar el itinerario si no vamos a emitir un ticket.
El comando para limpiar el pnr es: XI

El gds nos informa que el itinerario ha sido limpiado, y el ticket ha sido voideado.
Veremos solo el nombre en el pnr.

Datos y Comandos importantes

Gestión de aéreos

Para poder tener una mayor rentabilitad en cada pedido es importante conocer el proceso correcto de las herramientas de cobro
 Cuando procesamos un cobro por nuestro carrito de admin, estamos pagando un costo por el uso de la plataforma bajo nuestro numero de comercio.
Este importe es porcentual, es decir cuanto mayor sea el importe que procesemos en el carrito, mayor es el costo. O sea, menos ganancia de esa reserva
La gestión entonces se debe realizar de la siguiente manera:
Se cobra por separado:
Monto aéreo ↠ con el comercio de la aerolínea
Cargos de gestión Atrápalo + Seguros/Asientos ↠ con el comercio de Atrápalo

¿Qué significa UATP?

Son los cobros que se realizan con el comercio de la compañía aérea
Dependiendo del tipo de tarjeta, se pueden cobrar directamente desde el GDS o mediante la pasarela de cobros habilitada por la companía aérea.
Al momento de procesar un cobro UATP, es súper importante verificar el número de comercio por donde se va a cobrar

La caja del admin siempre registrará lo cobrado con comercio Atrápalo, es decir el fee de gestión Atrápalo, seguros, Asistencia web, asientos, servicios, etc.

No es posible avanzar con UATP
En algunas ocasiones podemos encontrarnos con compañias aereas que no tienen numero de comercio para procesar ciertos tipos de tarjetas.
O bien que solo permitan la emisión de tickets CASH.
Si este escenario se presenta vamos a cobrar por nuestra comercio a traves de la caja del admin y procederemos a pagar CASH el importe de los tickets .
En la caja de cobros va a figurar el total de la reserva a pagar por el pasajero.
